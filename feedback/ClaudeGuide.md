## PART 1: LOCKED EDITORIAL PREFERENCES (All 12 Chapters)

Apply these preferences uniformly across every chapter.

### VOICE & TONE

**Standard:** Authoritative + conversational (confident, expert, human)

‚úì **DO:**
- Make confident assertions backed by evidence
- Use active voice ("you must implement" not "must be implemented")
- Short, punchy sentences for emphasis
- Specific observations over generic principles
- Examples that demonstrate thinking, not just illustration

‚úó **DON'T:**
- Hedge language: "might," "could," "arguably," "perhaps," "some might suggest"
- Passive voice: "must be considered," "should be designed," "is believed that"
- Generic language: "can be said that," "one might argue," "in many cases"
- Weak setups: "provides compelling evidence," "validates that"
- Qualifying modifiers: "somewhat," "relatively," "fairly," "rather"

**Example:**

**BEFORE (Weak):**
```
"It could be argued that infrastructure might be considered important for agent deployment. 
Some research suggests this approach could potentially improve outcomes."
```

**AFTER (Confident):**
```
"Infrastructure is foundational to agent success. Every failed pilot traces back to infrastructure gaps."
```

---

### NARRATIVE STRUCTURE

**Standard:** Story/consequence first, then explanation

‚úì **Lead with:**
- Human moment (Maria typing into the scheduler)
- Consequence (adoption dropped to 8%)
- Question (Why did 95% fail?)
- Tension (Sarah's 90-day ultimatum)

‚úì **Then add:**
- Explanation of what happened
- Why it matters
- What to do about it

‚úó **Don't:**
- Start with definition ("Trust is the foundation...")
- Start with framework ("The INPACT‚Ñ¢ framework defines...")
- Start with abstract principle
- Follow story with explanation (reader already infers)

**Example:**

**BEFORE (Definition first):**
```
"The Instant dimension defines the requirement that agents must respond within two seconds. 
This is critical because slow responses cause user abandonment. Maria Rodriguez's experience 
demonstrates this."
```

**AFTER (Story first):**
```
"Maria's scheduling agent took nine to thirteen seconds to respond. That's how long it takes 
to pick up the phone and book manually. Adoption: 8%. That's what happens when infrastructure 
can't deliver instant responses."
```

**Standard:** Less is more. Framework names should be background, not foreground.

**INPACT‚Ñ¢ Target:** 35-40 mentions per chapter (not 70-90)
- First mention: "INPACT‚Ñ¢ framework" or "INPACT‚Ñ¢‚Äîsix fundamental needs"
- Remove ‚Ñ¢ symbol after first mention
- Subsequent: Use pronouns ("it," "these needs"), paraphrasing, or describe without naming

**"Architecture" Target:** 4-5 mentions per chapter (not 8-12)
- First mention: "7-Layer Architecture" (full name, define it)
- Subsequent: "the architecture," "this framework," "the layers"

**Example Reduction:**

**BEFORE (8 mentions in 3 sentences):**
```

INPACT‚Ñ¢ requirements. When infrastructure fulfills INPACT‚Ñ¢ needs, agents work. This INPACT‚Ñ¢

**AFTER (1 mention):**

```

### NUMBERS & PRECISION

**Standard:** Research = precise; Illustrative = rounded

**Research Statistics (KEEP PRECISE):**
- "3,613 employees and 238 C-level executives"
- "57% cite security and compliance"
- "Deloitte's Q3 2025 report"
- Credibility signal: precise = researched

**Illustrative Examples (ROUND):**
- "237 new physicians added" ‚Üí "dozens of new providers"
- "18 new procedure codes" ‚Üí "expanded services"
- "9-13 seconds" ‚Üí "nine to thirteen seconds" (spell out) or "roughly 10 seconds"
- "8-24 hour lag" ‚Üí "overnight refresh" or "daily batch"

**Spell Out:**
- Single digits: "nine seconds," not "9 seconds"
- Round numbers: "thirty days," "two weeks," "six months"
- Keep: "2 AM" (time), "Q3 2025" (date), "$2 million" (specific amount)

### SCAFFOLDING LANGUAGE

**Standard:** Remove entirely

‚úó **Delete:**
- "As Chapter X established..."
- "As we've covered..."

- "The INPACT‚Ñ¢ framework defines..."

### REPETITION RULES

**Standard:** Explain once thoroughly, reference elsewhere

**Pattern:**
1. First mention: Full explanation (definition + why + how + consequence)
2. Second mention: Reference with link to first explanation
3. Subsequent: Use as needed, don't re-explain

**Example:**

**FIRST MENTION:**
```
"The Instant dimension means sub-2-second response times. Users abandon slower systems‚Äînot 
because the AI is wrong, but because they don't wait. This requires real-time data fabric (Layer 2 
of the 7-Layer Architecture), which we detail in Chapter 3."
```

**SECOND MENTION:**
```
"The Instant dimension (Chapter 2) drove adoption from 60% target to 8% actual‚Äîusers won't wait."
```

**SUBSEQUENT:**
```
"Instant response times are non-negotiable."
```

## PART 2: THE 10 KEY ISSUE TYPES

These patterns appear repeatedly across chapters. Learn to spot and fix them.

### Issue 1: Framework Repetition (INPACT‚Ñ¢, Architecture, Dimension Names)

**Severity:** CRITICAL
**Pattern:** Framework/dimension name appears 3+ times in short passage
**Fix:** Find/Replace, manual review, use pronouns

**Detect:**
```
Count: "INPACT‚Ñ¢" (target <40/chapter)
Count: "Architecture" (target <5/chapter)
Count: "dimension" (check if redundant)
```

**Quick Fix:** Search for framework names, delete every other mention, replace with pronouns

**Time:** 1-2 hours per chapter

---

### Issue 2: Scaffolding Language

**Severity:** HIGH
**Pattern:** "As established," "The framework defines," "This requires," "must be"
**Fix:** Delete or reword to active voice

**Common Phrases:**
- "As we've established..." ‚Üí DELETE
- "The INPACT‚Ñ¢ framework defines..." ‚Üí Reword: "Six needs define..."
- "This dimension requires..." ‚Üí Reword: "You need..."
- "must be implemented" ‚Üí "you must implement"

**Quick Fix:** Search these exact phrases, rewrite

**Time:** 1-1.5 hours per chapter

---

### Issue 3: Interactive Elements & Emojis

**Severity:** MEDIUM-HIGH

### Issue 4: Dimension/Section Structure Formula

**Severity:** HIGH
**Pattern:** All dimensions explained identically (definition ‚Üí why ‚Üí technical ‚Üí business impact)
**Fix:** Vary each dimension's entry point and emphasis

**Detect:** 
```
Do all dimensions start the same way? 
Do all have identical structure?
Do all end with assessment rubric?
```

**Fix Strategy:**
- Dimension 1: Lead with consequence, then story, then definition
- Dimension 2: Lead with technical constraint, then business impact
- Dimension 3: Lead with business outcome, then technical requirement
- Etc. (vary all 6+)

**Time:** 1.5-2 hours per chapter

---

### Issue 5: Over-Explanation (Concepts Explained 3+ Times)

**Severity:** HIGH
**Pattern:** Concept introduced, then re-explained, then applied, then reference in diagram
**Fix:** Explain once; reference elsewhere

**Detect:**
```
Is "real-time data" explained more than once?
Is "RBAC + ABAC" explained multiple times?
Is same example retold across sections?
```

**Fix:** Keep thorough explanation in first instance; use 1-2 sentence reference elsewhere

**Time:** 1-2 hours per chapter

---

### Issue 6: "The X Pattern" Titles

**Severity:** MEDIUM
**Pattern:** All section headers follow "The X Pattern" (The Crisis, The Reality, The Solution)
**Fix:** Vary headers with different entry points

**Detect:**
```
Count headers starting with "The..."
Are they formulaic?
```

**Fix Examples:**

| Instead Of | Change To |
|---|---|
| "The Instant Dimension" | "Speed = User Confidence" |
| "The Natural Barrier" | "When Language Breaks Trust" |
| "The Commitment Paradox" | "Why Success Metrics Lie" |

**Time:** 45 minutes per chapter

---

### Issue 7: Passive Voice in Recommendations

**Severity:** MEDIUM
**Pattern:** "must be implemented," "should be designed," "is required"
**Fix:** Use active voice with clear actor

**Detect:**
```
Search: " is "
Search: " must be "
Search: " should be "
Count instances where no actor is clear
```

**Fix:**
- "must be implemented" ‚Üí "you must implement"
- "infrastructure should be designed" ‚Üí "design infrastructure to..."
- "agents are required to" ‚Üí "agents must..."

**Time:** 1 hour per chapter

---

### Issue 8: Jargon Without Plain Language First

**Severity:** MEDIUM

**Time:** 1 hour per chapter

**Do after Quick Wins:**

   - Rewrite each with different entry point
   - Verify each is unique from others

   - Find concepts explained 3+ times
   - Keep thorough explanation first time
   - Reference elsewhere (1-2 sentences)

## PART 9: COMMON MISTAKES TO AVOID

**Don't:**
1. Over-correct by removing necessary pedagogy
2. Delete all repetition (some structural repetition aids learning)
3. Change chapter structure or reorder major sections
4. Lose important technical content in pursuit of brevity
5. Rewrite opening 5+ times (do it once, move on)
6. Try to fix everything simultaneously (use phases: quick wins ‚Üí deep work ‚Üí polish)

**Do:**
1. Delete redundant explanation (when same concept explained 3+ times)
2. Keep structural repetition with voice variation
3. Keep chapter structure intact
4. Keep technical rigor while improving voice
5. Rewrite major sections 1-2 times maximum
6. Follow the phase approach for consistency and speed
